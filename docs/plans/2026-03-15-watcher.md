# Watch folder and slideshow.yml for live updates

> Automatically detect changes to images in the slideshow folder and the slideshow.yml file, updating the app state in real time without manual reload.

## Status

- **Phase:** Draft
- **Type:** feature

## Changelog

- Slideshow automatically picks up new, renamed, moved, or deleted images without reloading
- Changes to `slideshow.yml` (title, metadata) are reflected live
- Sidecar `.md` file edits update captions and presenter notes in place

## Motivation

Currently the app loads a folder once and builds its slide list from that snapshot. If a user adds, removes, or reorders images in Finder — or edits `slideshow.yml` or sidecar files externally — they must close and reopen the slideshow to see changes. A file watcher makes the app feel alive: edits in Lightroom, Finder drag-reorders, or YAML tweaks in a text editor show up instantly.

## Motivation

First step of the UI overhaul following the data-format change. The new `slideshow.md` format (PR #10) makes the entire slideshow a single markdown document. Users need to see this document inside the app — for reviewing, checking structure, and verifying content before presenting. List and grid show slides as discrete items; text view shows them as a continuous document.

## Design

### Approach

Add `.text` to the existing `ViewMode` enum (`.list`, `.grid`, `.text`). The toolbar picker gets a third segment (SF Symbol `doc.plaintext`). When selected, `SlideListPanel` renders a new `SlideshowTextView` that serializes the in-memory `SlideshowDocument` via `SlideshowWriter` and displays it as read-only monospaced text in a `ScrollView`.

Key decisions:
- **In-memory serialization** — uses `SlideshowWriter.write()`, not file reads, so text always reflects unsaved edits
- **Read-only** — no editing; users edit via EditorPanel or external editor
- **No slide selection sync** — clicking in text does not select a slide (future enhancement)
- **Text selection enabled** — users can copy snippets

Full spec: [`docs/superpowers/specs/2026-03-15-text-view-design.md`](../superpowers/specs/2026-03-15-text-view-design.md)

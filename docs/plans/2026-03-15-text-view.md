## Motivation

UI overhaul following the data-format change. The new `slideshow.md` format (PR #10) makes the entire slideshow a single markdown document. Editing the document directly in a text view is more natural than filling in separate caption/source/notes fields in an inspector panel. List and grid modes remain for visual reordering; the text view becomes the primary editing interface.

Subsumes the remove-inspector plan (PR #18, closed).
Supersedes panel-layout-fixes plan (PR #1, closed).

## Design

### Layout change

Before (3 panes + inspector):
```
PreviewPanel | SlideListPanel | Inspector (EditorPanel + FileInfoPanel)
```

After (2 panes, no inspector):
```
PreviewPanel | SlideListPanel (list / grid / text)
```

The toolbar loses the inspector toggle button and its Cmd+Opt+I shortcut.

### Editable text view

Add `.text` to `ContentView.ViewMode` enum. The toolbar picker gets a third segment (SF Symbol `doc.plaintext`, picker width ~120pt).

When `.text` is selected, `SlideListPanel` renders `SlideshowTextView` ŌĆö a `TextEditor` showing the full `slideshow.md` content in a monospaced system font.

**Text Ōåö model synchronization:**

The text view maintains its own `@State` text buffer. This is the source of truth while editing. The model is updated only on save, not on every keystroke. This avoids:
- Expensive mid-edit re-parsing
- Cursor position jumps from model updates
- Conflicts between text edits and observation-driven view refreshes

**Entering text mode:** `SlideshowWriter.write(document)` serializes the current model state into the text buffer.

**Leaving text mode (switching to list/grid):** If the text is dirty, save first (parse ŌåÆ update model ŌåÆ write to disk), then switch. List/grid views reflect the freshly parsed model.

**Structural operations in list/grid mode** (reorder, add, remove) modify the model and call `save()` as they already do. Switching to text mode regenerates from the model.

### TextEdit-style saving

Save behavior modeled after TextEdit.app, without adopting `NSDocument`:

1. **Cmd+S** ŌĆö explicit save via `.keyboardShortcut("s")`
2. **Auto-save on deactivation** ŌĆö `NSApplication.willResignActiveNotification`
3. **Auto-save on window close** ŌĆö handled by deactivation or app termination
4. **Dirty-state indicator** ŌĆö `NSWindow.isDocumentEdited` shows the dot in the close button

Save flow:
1. Take current text from TextEditor
2. Parse via `SlideshowParser` ŌåÆ `SlideshowDocument`
3. Update `slideshow.document` and `slideshow.slides` from parsed result
4. Write to disk via `SlideshowWriter`
5. Clear dirty state

**Parse errors:** `SlideshowParser` is lenient ŌĆö any text between `---` separators becomes a slide section. Unrecognized content is preserved in `SlideSection.unrecognizedContent`. The raw text is always saved to disk as-is (it's valid markdown even if it doesn't round-trip perfectly). If parsing produces unexpected results, the user sees the effect in list/grid views and can correct it.

### Dirty state tracking

Compare current TextEditor content against the text that was last loaded/saved. Simple string equality ŌĆö no structural diffing needed. Dirty state drives:
- `NSWindow.isDocumentEdited` (title bar dot)
- Save-on-deactivation (only saves if dirty)
- View mode switch (only parses if dirty)

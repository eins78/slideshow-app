# Support plain folders with optional slideshow.yml project file

> Drop the `.slideshow` bundle requirement. Any folder of images is a valid project. An optional `slideshow.yml` project file adds title and future layout metadata.

## Changelog

- Open any folder of images as a slideshow — no special bundle format required

## Motivation

The app currently requires `.slideshow` bundles (folders with `com.apple.package` UTType). This adds friction: users can't point at an existing photo folder and present. The working title "Mappe" (German for folder/portfolio) reinforces the core insight — a slideshow project IS a folder.

The `.slideshow` bundle adds no real value internally. FolderScanner, FileReorderer, BookmarkManager, and all models already treat it as a plain directory. The bundle is cosmetic (Finder icon) and limiting (can't open regular folders).

## Design

### Approach

1. **MANIFESTO.md** — establish design principles and 8-question checklist as design authority
2. **ProjectFile model + parser + writer** in SlideshowKit (TDD, follows SidecarParser/Writer pattern)
3. **FolderScanner integration** — new `ScanResult` return type with optional `ProjectFile`
4. **Slideshow model** — add `projectFile` property, derive `name` from project file title
5. **App layer** — simplify to `allowedContentTypes: [.folder]`, remove UTType declaration
6. **Examples** — rename from `.slideshow` bundles to plain folders, add `slideshow.yml`

### Key decisions

- **Filename:** `slideshow.yml` (visible, YAML, human-editable, serves as project marker)
- **Schema v1:** `version: 1` + optional `title` — minimal, unknown keys preserved
- **File order unchanged:** `003--` prefix convention stays, project file does NOT replace ordering
- **Graceful degradation:** no project file = folder name as title, no sidecar = image-only slide
